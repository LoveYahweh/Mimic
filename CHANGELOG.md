# Changelog

All notable changes to Mimic are documented here. This project follows
[Semantic Versioning](https://semver.org).

## 1.1.0

### Added
- **Argument-matched stubs.** Each non-void method gains `…When(_ match:, return:)` and
  `…When(_ match:, perform:)`, registering predicate-keyed stubs that are tried in order
  before the handler — so different inputs can return different results. Cleared by
  `mimicReset()`. Skipped for generic / `Self` / typed-throws returns and zero-parameter
  methods.

## 1.0.0

First stable release. `@Mockable` generates a `Mock<Protocol>` test double with zero
third-party runtime dependencies (`swift-syntax` is compile-time only). Verified with 82
package tests and an Xcode iOS app test target on the simulator.

### Requirement kinds
- **Methods** — `sync` / `async` / `throws` / typed `throws`, and **generic** methods
  (type-erased to `Any`, force-cast back; generic + `where` clauses preserved).
- **Properties** — get-only / get-set / optional / collection / function-type, with
  default-nil behaviour where applicable.
- **Subscripts** — get and get-set, multi-parameter, and overloaded; separate
  `…GetHandler` / `…SetHandler` and call recording (the setter records the new value).
- **`static`**, **`mutating`**, and **`nonisolated`** requirements.
- **Overloads** disambiguated by label, arity, type, or async-ness.

### Parameters & types
- `inout`, variadic (`Int...`), `borrowing`/`consuming`, closure, `@escaping`,
  `@autoclosure`, tuple, defaulted, and keyword-named parameters.
- Every type form: optionals, IUO (`Int!`), nested optionals, arrays / dictionaries /
  sets, tuples, function types, existentials (`any P`), compositions (`A & B`), metatypes
  (`T.Type`), key paths, nested generics, and `Self` (result and parameter).

### Concurrency
- `@MainActor` and custom global-actor protocols (isolation propagated to the mock and its
  handler closures), including `nonisolated` members reachable off the actor.

### Stubbing & recording
- `…Handler` closures, `…ReturnValue` constant stub, `…Returns(x, y, z)` sequential stub,
  `…ThrowsError(_:)` for throwing requirements, and `mimicReset()`.
- `…CallCount`, `…Calls` (labelled tuples for multi-parameter members), `…WasCalled`,
  `…LastCall`. Methods returning `Optional`/`Array`/`Dictionary`/`Set` default to an empty
  value when unstubbed.

### Other
- Access-level mirroring (`public`/`package` protocols get matching mocks).
- Clear diagnostics for requirements that can't be generated (`init`, `associatedtype`,
  operators) instead of confusing conformance errors.
- A worked `MimicDemo` SPM example and an `XcodeDemo/` iOS app, both fully tested.
- GitHub Actions CI running `swift test` on macOS.

### Known limitations
- `rethrows`, `init`, and `associatedtype` requirements, operator requirements, and
  effectful property accessors aren't generated (the first four emit a clear warning).
- Inherited / composed protocol requirements can't be generated by a peer macro — it only
  sees the annotated protocol's own syntax, never the parent's members.
- A non-escaping closure hidden behind a `typealias` can't be detected (a macro can't
  resolve the alias).
