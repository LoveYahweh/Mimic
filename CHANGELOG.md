# Changelog

All notable changes to Mimic are documented here. This project follows
[Semantic Versioning](https://semver.org).

## 1.0.0

First stable release. `@Mockable` generates a `Mock<Protocol>` test double with zero
third-party runtime dependencies (`swift-syntax` is compile-time only).

### Mocking surface
- Per-member `…Handler` stub closures, `…CallCount`, `…Calls` (labelled tuples for
  multi-parameter methods), `…WasCalled`, and `…LastCall`.
- `sync` / `async` / `throws` / typed `throws` methods.
- Generic methods (type-erased to `Any`, force-cast back; generic and `where` clauses
  preserved).
- `inout`, variadic, closure, tuple, optional, and defaulted parameters.
- `mutating` and `static` requirements; overloads by label, arity, type, or async-ness.
- Get-only / get-set / optional / collection properties.
- Methods returning `Optional`/`Array`/`Dictionary`/`Set` default to an empty value when
  unstubbed.

### Stubbing ergonomics
- `…ReturnValue` constant stub, `…Returns(x, y, z)` sequential stub, and `…ThrowsError(_:)`
  for throwing requirements.
- `mimicReset()` returns a mock to a clean slate.

### Other
- Access-level mirroring (`public`/`package` protocols get matching mocks).
- A worked `MimicDemo` SPM example and an `XcodeDemo/` iOS app, both fully tested.
- GitHub Actions CI running `swift test` on macOS.

### Known limitations
- `subscript`, `rethrows`, `init`/`associatedtype` requirements, and effectful property
  accessors are not generated.
- Inherited / composed protocol requirements can't be generated by a peer macro (it only
  sees the annotated protocol's own syntax).
